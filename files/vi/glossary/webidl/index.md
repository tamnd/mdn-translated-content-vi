**WebIDL** là ngôn ngữ mô tả giao diện được sử dụng để mô tả {{Glossary("type", "kiểu dữ liệu")}}, {{Glossary("interface", "giao diện")}}, {{Glossary("method", "phương thức")}}, {{Glossary("property", "thuộc tính")}}, và các thành phần khác tạo nên giao diện lập trình ứng dụng web ({{Glossary("API")}}). Nó sử dụng cú pháp cách điệu độc lập với bất kỳ ngôn ngữ lập trình cụ thể nào, để mã nguồn được sử dụng để xây dựng mỗi API có thể được viết bằng ngôn ngữ phù hợp nhất, trong khi vẫn có thể ánh xạ các thành phần của API sang các cấu trúc tương thích JavaScript.

WebIDL được sử dụng trong hầu hết mọi {{Glossary("specification")}} API cho Web, và nhờ định dạng và cú pháp chuẩn của nó, các lập trình viên tạo ra trình duyệt web có thể dễ dàng đảm bảo rằng các trình duyệt của họ tương thích với nhau, bất kể họ chọn cách viết mã để triển khai API như thế nào.
